Where each category of personal data physically lives in the Wren database, with live row counts. This is the "records map" an inspector or the ICO may ask for.

Request a secure off-site archive

Archives records that are past their active-need window but still within legal retention. They are encrypted with gpg AES256 on this hardware before leaving the box, shipped to encrypted off-site storage, and a SHA-256 recorded. Nothing is deleted from the live database — hot-DB removal needs explicit sign-off.

Archive log

Parent requests for a copy of their child's data (subject access / portability) or for erasure (right to be forgotten). Approving an access request produces the readable memory package plus a machine-readable export. Approving an erasure request removes personal data while retaining legally-required records (safeguarding, accident/medication, financial, attendance) per the retention schedule. Every step is written to the audit log.
